# Project Summary

The Automated Loan Decisioning System (ALDS) project modernizes NorthRiver Bankâ€™s personal lending process by replacing manual underwriting steps with an automated, rules-driven solution.

---

## 1. Goals

- Reduce loan decision times from days to under an hour.  
- Improve consistency of lending decisions.  
- Enhance customer experience through transparency.  
- Strengthen compliance and audit readiness.

---

## 2. Deliverables

The project produced:

- Business and non-functional requirements.  
- User stories and Gherkin scenarios.  
- As-is and to-be workflow definitions.  
- Business rules catalogue.  
- Functional specification (FSD).  
- UAT test plan.  
- Impact analysis.  
- Diagrams (architecture, BPMN, ERD, state machine, sequence, UI).  
- Example BI dashboards and supporting SQL.  
- A consolidated PDF: `Requirement-Gathering.pdf`.

---

## 3. Outcomes

- A clear target operating model for automated lending.  
- Structured documentation suitable for stakeholder review.  
- A realistic portfolio artifact demonstrating end-to-end business analysis capabilities in a banking context.

---

## 4. How to Read This Project

- Start with Business Requirements and As-Is/To-Be workflows.  
- Review Business Rules and the FSD for system behavior.  
- Examine UAT and Impact Analysis for testing and stakeholder implications.  
- Open the diagrams and dashboards for visual understanding of the solution.
